# Prototype - Storywriting feedback + game skeleton
- Main focus:
	- Get feedback on the story direction in terms of narrative flow, choices, endings, etc...
	- Artistic/graphic design ideas for UI are welcomed, but not the focus
	- Create a skeleton that determines workload for artists & music composers
		- Developers can work on game architecture, Ren'Py transitions, UI layouts, etc...
- Storyline is entirely scripted, with Ren'Py logic implemented
	- Writing is rough/choppy/cringe but enough to get the point across; heavy refinement & cleaning for developer notes needed for flow/narrative
	- CG concept art should appear where they are expected for the final version
	- Basic transitions like shaking, being hit by something, earthquakes, etc... should be in
		- The transition itself can be refined later on; as long as the idea is there
	- Functional flow logic for story elements like route locking, intimacy point thresholds, and ending funneling
- All major features should be implemented
	- CG gallery
	- Character profiles
	- Title screen updates after each ending
- All art concepts included; there should be enough to know what needs to be drawn/commissioned and for what purpose
	- Colored with rough lineart is okay; but BGs/CGs and character designs should be 80~90% set in stone at least
	- Base character sprites are enough; no need for final emotions/poses implemented yet
	- Initial decisions for UI in terms of character sprite in corner, dialogue boxes, menus, etc...
	- Basic font choice is enough; no need to finalize at this stage
- Voice acting drafts (to help determine voice acting script & get a vibe for the game with voice acting)
	- Gotta decide how & where I'm going to find VAs
	- Feature to turn off voice acting if desired
- Music & sound effects are not necessary but would be nice to have
